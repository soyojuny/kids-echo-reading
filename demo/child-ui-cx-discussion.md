# 아동용 리딩 화면 CX 토의안

## 1) 목적
- 개발 착수 전에 아동용 핵심 경험(선택/읽기/피드백) 화면 구조를 고정한다.
- 데모 화면 기준으로 실제 API 연동 및 컴포넌트 분해를 진행한다.

## 2) 화면 구성 (현재 데모)
- `책 고르기`: 큰 카드 + 카테고리 칩 + 제목 검색.
- `읽기 연습`: 상단 삽화(1페이지/2페이지 보기 토글) + 하단 한 문장 표시.
- `문장 진행`: 짧은 단어 묶음(2토큰) 기준으로 `듣기 -> 따라 읽기` 자동 실행.
- `자동 이동 규칙`: 문장 읽기 성공 시 2초 후 자동 다음 문장, 실패 시 자동 이동 없음(수동 버튼으로만 이동).
- `페이지 이동 규칙`: 현재 1p/2p 뷰 범위 문장을 모두 읽으면 5초 후 자동 다음 페이지.
- `페이지 복습`: 페이지 완료 시 남은 단어가 있으면 `이 단어 한 번 더 해볼까?` 패널로 즉시 재도전.
- `최종 화면`: 점수 대신 `AI 추천 복습 단어` 목록 제공.
- `결과 보기`: 점수, 어려운 단어, 재시도 CTA.

## 3) 토의 포인트
- 대량 카탈로그: 카테고리 체계(대분류/세부분류)와 정렬 기준(최근/인기/난이도) 확정.
- 검색 UX: 초성/자동완성 필요 여부, 오타 허용 여부.
- 읽기 진행: 문장 단위 고정 + 문장 길이 제한(예: 12~18어절) 기준 필요.
- 읽기 중 피드백: 단어별 배지 대신 문장 상태 색상 변화(듣기/녹음/완료) 중심으로 단순화.
- 자동 진행 제어: 자동 재생 속도/대기 시간, 중지 후 재개 방식 합의 필요.

## 6) 대기 시간 추천
- 문장 성공 후 자동 다음 문장: `2.0초` (현재 데모 적용).  
- 페이지 완료 후 자동 다음 페이지: `5.0초` (현재 데모 적용).  
- 운영 추천 범위:
- 문장 이동은 `1.6~2.2초`, 페이지 이동은 `4~6초`에서 연령대별 A/B 테스트 권장.
- 부모/교사 관여: 아동 화면에 성과 수치를 어디까지 노출할지.

## 4) UI 확정 시 개발 기준
- 필수
- 책 목록 조회 API 연동.
- 페이지 자산(이미지, 텍스트, TTS 메타) 로딩.
- 녹음 업로드 및 발음 평가 결과 맵핑.
- 단어 단위 색상 상태 렌더링.
- 옵션
- 캐릭터 가이드/응원 애니메이션.
- 스티커 보상, 연속 학습 배지.
